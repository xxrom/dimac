# Security Policy

## Reporting A Vulnerability

Do not open a public issue for a security-sensitive problem.

Use GitHub private vulnerability reporting if it is enabled for the repository. If that is not available, contact the maintainer privately and include:

- affected version or commit
- reproduction steps
- impact assessment
- any proof-of-concept details needed to validate the report

## Scope

Security issues include:

- privilege or permission bypasses
- unsafe command execution paths
- data exposure beyond the local machine
- unexpected persistence of sensitive local state

## Supported Versions

Until the project has tagged releases, treat the default branch as the only supported version.
