# Code Of Conduct

## Expected Behavior

- Be respectful and direct.
- Focus criticism on code, behavior, and decisions rather than people.
- Assume good intent, then clarify disagreements with concrete technical detail.
- Avoid harassment, discrimination, intimidation, and personal attacks.

## Maintainer Standards

Project maintainers may edit, remove, or reject comments, issues, pull requests, and other contributions that make collaboration unsafe or unproductive.

## Reporting

If a discussion or contribution crosses the line, use GitHub reporting tools or contact the maintainer privately before escalating the discussion in public.
